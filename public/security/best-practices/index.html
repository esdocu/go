Mejores prácticas de seguridad para desarrolladores de Go

Volver a Seguridad en Go

Esta página proporciona a los desarrolladores de Go las mejores prácticas para priorizar la seguridad de sus proyectos. Desde automatizar tests con fuzzing hasta verificar fácilmente las condiciones de carrera, estos consejos pueden ayudar a que tu código base sea más seguro y confiable.

Escanea el código fuente y los binarios en busca de vulnerabilidades

Escanear periódicamente tu código y archivos binarios en busca de vulnerabilidades ayuda a identificar posibles riesgos de seguridad de manera temprana. Puedes usar govulncheck, respaldado por la base de datos de vulnerabilidades de Go para escanear tu código en busca de vulnerabilidades y analizar cuáles realmente te afectan. Comienza con el tutorial de govulncheck ⬀.

Govulncheck también se puede integrar en flujos de CI/CD. El equipo de Go proporciona una GitHub Action para govulncheck en GitHub Marketplace. Govulncheck también admite un indicador -json para ayudar a los desarrolladores a integrar el escaneo de vulnerabilidades con otros sistemas CI/CD.

También puedes buscar vulnerabilidades directamente en tu editor de código usando la Extensión Go para Visual Studio Code. Comienza con este tutorial ⬀.

Mantén actualizada tu versión de Go y tus dependencias

Mantener tu Versión Go actualizada ⬀ ofrece acceso a las últimas funciones del lenguaje, mejoras de rendimiento y parches para vulnerabilidades de seguridad conocidas. Una versión actualizada de Go también garantiza la compatibilidad con versiones más recientes de dependencias, lo que ayuda a evitar posibles problemas de integración. Revisa el Historial de lanzamientos de Go ⬀ para ver qué cambios se han realizado en Go entre lanzamientos. El equipo de Go emite lanzamientos puntuales a lo largo del ciclo de lanzamiento para solucionar errores de seguridad. Asegúrate de actualizar a la última versión menor de Go para asegurarse de tener las últimas correcciones de seguridad.

Mantener dependencias de terceros actualizadas también es crucial para la seguridad, el rendimiento y el cumplimiento del software con los últimos estándares en el ecosistema Go. Sin embargo, actualizar a las últimas versiones sin una revisión exhaustiva también puede ser riesgoso, ya que podría introducir nuevos errores, cambios incompatibles o incluso código malicioso. Por lo tanto, si bien es esencial actualizar las dependencias para obtener los últimos parches y mejoras de seguridad, cada actualización debe revisarse y probarse cuidadosamente.

Testing con fuzzing para descubrir edge-case exploits

Fuzzing ⬀ es un tipo de testing que utilizan una guía de cobertura para manipular entradas aleatorias y recorrer el código para encontrar e informar vulnerabilidades potenciales como inyecciones de SQL, desbordamientos de búfer, denegación de servicio y ataques de cross-site scripting. Fuzzing a menudo puede llegar a casos extremos que los programadores pasan por alto o consideran demasiado improbables para probar. Comienza con este tutorial ⬀.

Verifica las condiciones de carrera con el detector de carrera de Go

Las condiciones de carrera ocurren cuando dos o más gorrutinas ⬀ acceden al mismo recurso simultáneamente y al menos uno de esos accesos es una escritura. Esto puede generar problemas impredecibles y difíciles de diagnosticar en tu software. Identifica posibles condiciones de carrera en tu código Go utilizando el software integrado detector de carreras ⬀, que puede ayudarte a garantizar la seguridad y confiabilidad de tus programas concurrentes. Sin embargo, el detector de carreras encuentra carreras que ocurren en tiempo de ejecución, por lo que no encontrará carreras en rutas de código que no se ejecuten.

Para usar el detector de carrera, agrega el indicador -race cuando ejecutes tus tests o crees tu aplicación, por ejemplo, go test -race. Esto compilará tu código con el detector de carreras habilitado e informará cualquier condición de carrera que detecte en tiempo de ejecución. Cuando el detector de carreras encuentre una carrera de datos en el programa, imprime un informe ⬀ que contiene seguimientos de pila para accesos en conflicto y pilas donde se crearon las rutinas involucradas.

Usa Vet para examinar construcciones sospechosas

El comando vet de Go es diseñado para analizar tu código fuente y señalar problemas potenciales que podrían no ser necesariamente errores de sintaxis, pero que podrían generar problemas durante el tiempo de ejecución. Estos incluyen construcciones sospechosas, como código inalcanzable, variables no utilizadas y errores comunes en torno a las rutinas. Al detectar estos problemas desde el principio durante el proceso de desarrollo, go vet ayuda a mantener la calidad del código, reduce el tiempo de depuración y mejora la confiabilidad general del software. Para ejecutar go vet para un proyecto específico, ejecuta:

go vet ./...

Suscríbete a golang-announce para recibir notificaciones sobre versiones de seguridad

Las versiones de Go que contienen correcciones de seguridad se anuncian previamente en la lista de correo de bajo volumen golang-announce@googlegroups.com. Si deseas saber cuándo están en camino las correcciones de seguridad para Go, suscríbete.
