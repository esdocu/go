Tutorial: Crear un módulo Go

Esta es la primera parte de un tutorial que presenta algunas características fundamentales del lenguaje Go. Si recién estás comenzando con Go, asegúrate de consultar el Tutorial: Introducción a Go, que presenta el comando go, módulos Go y código Go muy simple.

En este tutorial crearás dos módulos. La primera es una biblioteca destinada a ser importada por otras bibliotecas o aplicaciones. La segunda es una aplicación de llamada que utilizará la primera.

La secuencia de este tutorial incluye siete temas breves, cada uno de los cuales ilustra una parte diferente del lenguaje.

  1. Crea un módulo -- escribe un módulo pequeño con funciones que puedas llamar desde otro módulo.
  2. Llama tu código desde otro módulo ⬀ -- Importa y usa tu nuevo módulo.
  3. Devolver y manejar un error ⬀ -- Agrega un manejo de errores simple.
  4. Devolver un saludo aleatorio ⬀ -- Manejar datos en slices (arrays de tamaño dinámico de Go).
  5. Devolver saludos para varias personas ⬀ -- Almacenar pares clave/valor en un map.
  6. Agregar una test ⬀ -- usa las funciones de test unitarios integradas de Go para realizar tests a tu codigo.
  7. Compila e instala la aplicación ⬀ -- Compila e instala tu código localmente.
Nota: Para otros tutoriales, consulta Tutoriales ⬀.

Requisitos previos

  • Alguna experiencia en programación. El código aquí es bastante simple, pero ayuda saber algo sobre funciones, bucles y arrays.
  • Una herramienta para editar tu código. Cualquier editor de texto que tengas funcionará bien. La mayoría de los editores de texto tienen buen soporte para Go. Los más populares son VSCode (gratis), GoLand (pago) y Vim (gratis).
  • Una terminal de comandos. Go funciona bien con cualquier terminal en Linux y Mac, y en PowerShell o cmd en Windows.

Inicia un módulo que otros puedan usar

Comienza creando un módulo Go. En un módulo, recopilas uno o más paquetes relacionados para un conjunto de funciones discreto y útil. Por ejemplo, podrías crear un módulo con paquetes que tengan funciones para realizar análisis financieros para que otros que escriban aplicaciones financieras puedan utilizar tu trabajo. Para obtener más información sobre el desarrollo de módulos, consulta Desarrollo y publicación de módulos ⬀.

El código Go se agrupa en paquetes y los paquetes se agrupan en módulos. Tu módulo especifica las dependencias necesarias para ejecutar su código, incluida la versión de Go y el conjunto de otros módulos que requiere.

A medida que agregas o mejoras la funcionalidad de tu módulo, publicas nuevas versiones del módulo. Los desarrolladores que escriben código que llaman a funciones en tu módulo pueden importar los paquetes actualizados del módulo y probar con la nueva versión antes de ponerlo en uso en producción.

  1. Abre un símbolo del sistema y haz cd a tu directorio de inicio.

    En Linux o Mac: 

    cd

    En Windows: 

    cd %HOMEPATH%
  2. Crea un directorio greetings para el código fuente de tu módulo Go.

    Por ejemplo, desde tu directorio personal utiliza los siguientes comandos: 

    mkdir greetings 
cd greetings
  3. Inicia tu módulo usando go mod init (comando) ⬀.

    Ejecuta el comando go mod init, dándole la ruta de tu módulo; aquí, usa example.com/greetings. Si publicas un módulo, debe ser una ruta desde la cual las herramientas de Go puedan descargar tu módulo. Ese sería el repositorio de tu código.

    Para obtener más información sobre cómo nombrar tu módulo con una ruta de módulo, consulta Administrar dependencias ⬀. 

    $ go mod init example.com/greetings
go: creating new go.mod: module example.com/greetings

    El comando go mod init crea un archivo go.mod para rastrear las dependencias de tu código. Hasta ahora, el archivo incluye solo el nombre de tu módulo y la versión de Go que admite tu código. Pero a medida que agregues dependencias, el archivo go.mod enumerará las versiones de las que depende tu código. Esto mantiene las compilaciones reproducibles y te brinda control directo sobre qué versiones de módulo usar.

  4. En tu editor de texto, crea un archivo en el que escribir tu código y llámalo greetings.go.
  5. Pega el siguiente código en tu archivo greetings.go y guárdalo. 
    package greetings

import "fmt"

// Hello returns a greeting for the named person.
func Hello(name string) string {
    // Return a greeting that embeds the name in a message.
    message := fmt.Sprintf("Hi, %v. Welcome!", name)
    return message
}

    Este es el primer código para tu módulo. Devuelve un saludo a cualquier llamador que lo solicite. Escribirás código que llame a esta función en el siguiente paso.

    En este código:

    • Declaraste un paquete greetings para recopilar funciones relacionadas.
    • Implementaste una función Hello para devolver el saludo.

      Esta función toma un parámetro name cuyo tipo es string. La función también devuelve un string. En Go, una función cuyo nombre comienza con una letra mayúscula puede ser llamada por una función que no esté en el mismo paquete. Esto se conoce en Go como nombre exportado. Para obtener más información sobre los nombres exportados, consulta Nombres exportados ⬀ en el tour de Go.

    • Declaraste una variable message para contener tu saludo.

      En Go, el operador := es un atajo para declarar e inicializar una variable en una línea (Go usa el valor de la derecha para determinar el tipo de variable). Tomando el camino más largo, podrías haber escrito esto como: 

      var message string
message = fmt.Sprintf("Hi, %v. Welcome!", name)
    • Utilizaste del paquete fmt la Sprintf (función) para crear un mensaje de saludo. El primer argumento es una cadena de formato y Sprintf sustituye el valor del parámetro name por el verbo de formato %v. Insertar el valor del parámetro name completa el texto del saludo.
    • Devolviste el texto de saludo formateado a el llamador.

En el siguiente paso, llamarás a esta función desde otro módulo.
